<objective>
Make privacy structural, not conventional. Add the physically-separate private namespace (`data/private.db` with its own key), expose it only through an explicit `get_private_factory()` (never a glob over `data/*.db`), provide an export-collection helper that excludes the private DB by construction, and lock secrets discipline (DB keys env-only, no secret literals in tracked files, Garmin token directory outside the repo). This satisfies PRIV-02 (separate file, excluded from export/telemetry by construction), PRIV-04 (secrets out of source/commits/logs; tokens outside repo), and deepens PRIV-01 (both stores encrypted via the same PRAGMA-key-first factory).

Purpose: A single missing WHERE clause or stray glob must NEVER be able to leak spiritual-practice data. File-level separation + an explicit-opt-in factory makes leakage structurally impossible, satisfying the locked "never a column flag" decision.
Output: dual-namespace factory, export-exclusion helper, and a green privacy/secrets test suite.
</objective>

| repo tree → committed artifacts | Tracked files must contain no secret values and no `.env`; token dir lives outside the tracked tree |
