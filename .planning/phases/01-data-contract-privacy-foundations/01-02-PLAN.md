<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| raw payload → stored JSON | Garmin/manual payload shapes validated by pydantic before becoming a stored `payload` JSON blob |
| schema_version → reader | Frozen version selects the correct parser; a mutated version mislabels stored rows |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Tampering | malformed/out-of-range stream payload | mitigate | pydantic models validate every stream + field_validator domain bounds (positive HRV, readiness 1..5); tests assert rejection |
| T-01-06 | Tampering | mutated `schema_version` on existing rows mislabels payloads | mitigate | schema_version is a frozen `Literal[1]`; Anti-Pattern documented; new versions = new classes, never UPDATE — covered by version-present test |
| T-01-07 | Repudiation | silent imputation hiding missing data | mitigate | all-Optional schemas + nullable `daily_facts.value`; missing-data-honesty test asserts None is preserved, never defaulted |
| T-01-SC | Tampering | pip/uv installs (supply chain) | accept | No NEW packages introduced in this plan (uses Plan 01's pinned, slopcheck-[OK] deps); no install task |
</threat_model>
